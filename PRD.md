## 文档信息
- **项目名称**：跨境电商3C电子类目每日热卖情报看板
- **文档版本**：v1.0
- **创建日期**：2026-03-14
- **产品经理**：AI Product Manager
- **目标用户**：跨境电商卖家、选品专员、电商运营人员

---

## 一、产品背景与目标

### 1.1 产品背景
跨境电商行业竞争激烈，卖家需要快速发现爆款商品以优化选品策略。3C电子类目是跨境电商的热门类目，产品更新快、价格敏感度高，卖家需要及时掌握市场动态。

### 1.2 产品目标
- 为跨境电商卖家提供3C电子类目在亚马逊、TikTok Shop、Shopee、Lazada四大平台的每日热卖商品数据
- 帮助卖家快速发现爆款、优化选品策略、提升销售业绩
- 提供直观、易用的数据展示界面，支持多维度筛选和搜索

### 1.3 成功指标
- 日活跃用户数 (DAU) > 1000
- 用户留存率 > 30%（7日）
- 页面平均加载时间 < 2秒
- 用户满意度 > 4.5/5

**场景2：竞品分析**

## 四、功能需求

### 4.1 功能列表

#### 4.1.1 平台切换功能
- **功能描述**：顶部导航栏显示4个平台选项（亚马逊、TikTok Shop、Shopee、Lazada），用户点击切换平台
- **交互逻辑**：
  - 默认显示亚马逊平台
  - 点击平台选项后，切换到对应平台的热卖榜单
  - 当前平台高亮显示
- **数据需求**：每个平台Top 100热卖商品

#### 4.1.2 热卖榜单功能
- **功能描述**：展示当日Top 100热卖3C电子商品，按销量降序排列
- **展示形式**：商品卡片网格布局（PC端4列，平板端3列，手机端2列）
- **数据需求**：商品ID、标题、图片、价格、销量、评分、店铺名、平台、商品URL

#### 4.1.3 商品卡片功能
- **功能描述**：展示单个商品的详细信息
- **卡片布局**：
  ```
  ┌─────────────────────┐
  │   [商品图片]        │
  │   [NEW] [↑]         │ ← 标签
  │   商品标题（2行）    │
  │   $29.99            │ ← 价格
  │   ⭐4.8 (1234)      │ ← 评分和评论数
  │   🔥销量: 5678      │ ← 销量
  │   店铺: ABC Store    │ ← 店铺名
  └─────────────────────┘
  ```
- **交互逻辑**：点击卡片跳转到商品详情页

#### 4.1.4 筛选功能
- **功能描述**：支持按价格、评分、销量筛选商品
- **筛选条件**：
  - 价格区间：全部、$0-$20、$20-$50、$50-$100、$100+
  - 评分：全部、4.0+、4.5+、4.8+
  - 销量：全部、100+、500+、1000+
- **交互逻辑**：
  - 筛选条件可以组合使用
  - 筛选后实时更新商品列表
  - 显示筛选结果数量

#### 4.1.5 搜索功能
- **功能描述**：支持关键词搜索商品
- **搜索范围**：商品标题、店铺名
- **交互逻辑**：
  - 输入关键词后按回车或点击搜索按钮
  - 搜索结果高亮显示匹配的关键词
  - 显示搜索结果数量
  - 支持清空搜索

#### 4.1.6 趋势标识功能
- **功能描述**：显示新品标识和销量趋势标识
- **标识类型**：
  - 新品标识：商品上架时间<7天，显示"NEW"标签
  - 销量上升标识：昨日销量比前日增长>20%，显示"↑"图标
  - 销量下降标识：昨日销量比前日下降>20%，显示"↓"图标
  - 销量稳定：无标识

#### 4.1.7 跳转购买功能
- **功能描述**：点击商品卡片跳转到对应平台的商品详情页
- **交互逻辑**：
  - 点击商品卡片在新标签页打开商品详情页
  - 支持右键菜单（在新标签页打开、复制链接等）

#### 4.1.8 数据时间显示功能
- **功能描述**：显示数据更新时间
- **展示位置**：页面底部
- **展示格式**："数据更新于：2026-03-14 02:00"

#### 4.1.9 响应式布局功能
- **功能描述**：支持PC、平板、手机端自适应
- **布局规则**：
  - PC端（>1024px）：4列网格
  - 平板端（768px-1024px）：3列网格
  - 手机端（<768px）：2列网格

#### 4.1.10 历史数据查看功能
- **功能描述**：支持查看近一周和近一个月的热卖商品数据
- **时间范围选择**：
  - 今日（默认）
  - 近一周
  - 近一个月
- **交互逻辑**：
  - 顶部添加时间范围选择器
  - 选择不同时间范围后，显示对应时期的热卖商品
  - 支持与平台切换、搜索、筛选功能组合使用
- **数据需求**：按日期存储的商品数据，包含历史销量、价格等信息

## 六、数据字段定义

### 6.1 商品数据模型

```typescript
interface Product {
  id: string;                    // 商品唯一ID
  title: string;                 // 商品标题
  image: string;                 // 商品图片URL
  price: number;                 // 当前价格（美元）
  originalPrice?: number;        // 原价（如有折扣）
  currency: string;              // 货币单位
  rating: number;                // 评分（0-5）
  reviewCount: number;           // 评论数
  salesCount: number;            // 销量（估算）
  shopName: string;              // 店铺名称
  platform: 'amazon' | 'tiktok' | 'shopee' | 'lazada';  // 平台
  productUrl: string;            // 商品详情页URL
  isNew: boolean;                // 是否新品（上架时间<7天）
  salesTrend: 'up' | 'down' | 'stable';  // 销量趋势
  category: string;              // 子类目（耳机、充电器等）
  listedDate: string;            // 上架日期
  updatedAt: string;             // 数据更新时间
}
```

### 6.2 数据来源

| 平台 | 数据来源 | 数据字段 |
|-----|---------|---------|
| **亚马逊** | Bright Data - web_data_amazon_product | title, price, rating, reviewCount, salesCount, shopName, productUrl |
| **TikTok Shop** | Bright Data - web_data_tiktok_product | title, price, rating, reviewCount, salesCount, shopName, productUrl |
| **Shopee** | Bright Data - web_data_shopee_product | title, price, rating, reviewCount, salesCount, shopName, productUrl |
| **Lazada** | Bright Data - web_data_lazada_product | title, price, rating, reviewCount, salesCount, shopName, productUrl |

### 6.3 数据更新策略

- **更新频率**：每日更新一次（凌晨2:00）
- **数据缓存**：Redis缓存24小时
- **数据清洗**：
  - 去重：同一商品在不同平台
  - 排序：按销量降序
  - 过滤：只保留3C电子类目商品
  - 取Top 100

## 十一、风险与应对

| 风险 | 影响 | 概率 | 应对方案 |
|-----|------|------|---------|
| 数据源不稳定 | 无法获取数据 | 中 | 配置多个备用数据源，增加重试机制 |
| 反爬机制触发 | IP被封 | 低 | 使用Bright Data代理池，控制抓取频率 |
| API成本超预算 | 运营成本高 | 中 | 使用Redis缓存减少调用，优化抓取策略 |
| 跨平台数据差异 | 数据格式不统一 | 中 | 设计统一数据模型，编写数据清洗逻辑 |
| 性能瓶颈 | 页面加载慢 | 低 | 图片懒加载，分页加载，CDN加速 |
